## 系统介绍

### 什么是队列看板？

队列看板系统是为质培团队打造的**实时质量监控与分析平台**，帮助你：

- ✅ **实时监控**：快速了解当日质检情况
- 📊 **错误分析**：发现问题类型和趋势
- 📈 **数据可视化**：查看性能趋势和变化
- 🎯 **精准定位**：找到需要重点关注的队列和审核人

### 核心价值

| 角色 | 价值 |
|-----|------|
| **团队管理者** | 全局质量监控、资源调配决策 |
| **质检员** | 了解自身表现、发现改进方向 |
| **培训同学** | 识别培训需求、跟踪培训效果 |

## 功能详解

### 1. 实时监控页面

**访问路径**: http://localhost:3000/monitor

**功能概述**: 查看当日 vs 昨日的质检核心指标对比

#### 页面组成

##### 1.1 核心指标卡片

显示4个关键指标：

| 指标 | 说明 | 示例 |
|-----|------|------|
| **正确率** | 当日正确数 / 质检总量 | 99.35% ↓ 0.03% |
| **质检总量** | 当日总审核条数 | 39,140 ↑ 9,369 |
| **误判率** | 误判数 / 质检总量 | 0.27% ↓ 0.01% |
| **申诉率** | 申诉数 / 质检总量 | 0.00% |

**颜色说明**:
- 🟢 绿色箭头 (↓): 误判率/申诉率下降（好）
- 🔴 红色箭头 (↑): 误判率/申诉率上升（差）
- 🟡 黄色箭头 (↑): 正确率/质检量上升（好）

##### 1.2 重点关注队列

显示**正确率 < 90%** 的队列，需要重点关注。

**字段说明**:
- 队列名称
- 审核量
- 正确率
- 误判率

**如果显示为空**: 说明所有队列正确率都 ≥ 90%，质量良好 ✅

##### 1.3 高频错误类型 Top 5

显示当日出现次数最多的错误类型。

**用途**: 
- 识别培训重点
- 调整审核规则
- 发现系统性问题

##### 1.4 重点关注审核人

显示**正确率 < 85%** 的审核人，需要进行针对性培训。

**字段说明**:
- 审核人姓名
- 审核量
- 正确率
- 误判率

#### 使用技巧

1. **日期切换**: 
   - 点击日期选择器选择日期
   - 点击"刷新数据"按钮加载

2. **数据对比**: 
   - 关注箭头方向和变化幅度
   - 大幅度变化需要深入分析

3. **异常识别**:
   - 正确率突然下降 > 5%
   - 误判率突然上升 > 1%
   - 某个队列/审核人表现异常

### 2. 错误分析页面

**访问路径**: http://localhost:3000/error-analysis

**功能概述**: 深入分析错误类型、趋势和根因

#### 页面组成

##### 2.1 错误总览

显示指定天数内的错误汇总：

| 指标 | 说明 |
|-----|------|
| **总错误数** | 时间范围内所有错误 |
| **总审核量** | 时间范围内所有审核 |
| **错误率** | 总错误数 / 总审核量 |
| **误判数** | 最终判定为误判的数量 |
| **误判率** | 误判数 / 总审核量 |
| **错误类型数** | 不同错误分类的数量 |

##### 2.2 天数选择器

支持3个时间范围：
- **最近7天** (默认): 查看一周趋势
- **最近14天**: 半月分析
- **最近30天**: 月度统计

##### 2.3 错误类型分布 Top 10

按错误次数排序，显示Top 10错误类型。

**字段说明**:
- **排名**: 1-10
- **错误类型**: 具体错误分类（如"违规引流"）
- **数量**: 出现次数
- **占比**: 占总错误的百分比
- **趋势**: 稳定/上升/下降

**示例数据**:
```
1. 违规引流  4次  0.52%  → 稳定
2. 低质导流  2次  0.26%  → 稳定
3. 政治敏感  1次  0.13%  → 稳定
```

## 数据指标说明

### 核心指标定义

| 指标 | 计算公式 | 合格标准 |
|-----|----------|---------|
| **正确率** | 正确数 / 质检总量 × 100% | ≥ 90% |
| **误判率** | 误判数 / 质检总量 × 100% | ≤ 5% |
| **申诉率** | 申诉数 / 质检总量 × 100% | ≤ 2% |
| **错误率** | 错误数 / 质检总量 × 100% | ≤ 10% |

### 数据来源

所有数据来自 **fact_qa_event** 表，字段说明：

| 字段 | 说明 | 示例值 |
|-----|------|--------|
| `biz_date` | 业务日期 | 2026-04-01 |
| `queue_name` | 队列名称 | 评论_A组 |
| `reviewer_name` | 审核人 | 张三 |
| `is_final_correct` | 是否最终正确 | 1/0 |
| `is_misjudge` | 是否误判 | 1/0 |
| `is_appealed` | 是否申诉 | 1/0 |
| `error_type` | 错误类型 | 违规引流 |

### 数据更新频率

- **开发环境**: 手动刷新
- **生产环境**: 每日凌晨自动更新（计划）

## 常见使用场景

### 场景1: 每日质量晨会

**使用页面**: 实时监控

**操作步骤**:
1. 访问 /monitor 页面
2. 选择昨日日期
3. 查看核心指标对比
4. 关注"重点关注队列"和"重点关注审核人"
5. 截图分享到会议

**关注重点**:
- 正确率是否达标 (≥ 90%)
- 误判率是否控制 (≤ 5%)
- 有无队列/审核人需要重点关注

---

### 场景2: 周度质量分析

**使用页面**: 错误分析

**操作步骤**:
1. 访问 /error-analysis 页面
2. 选择"最近7天"
3. 查看错误总览和错误类型分布
4. 导出截图用于周报

**分析维度**:
- 本周总错误数 vs 上周
- Top 3 高频错误类型
- 错误趋势（上升/下降）

---

### 场景3: 月度性能评估

**使用页面**: 数据可视化

**操作步骤**:
1. 访问 /visualization 页面
2. 选择"最近30天"
3. 查看性能趋势总览
4. 分析正确率趋势图

**评估维度**:
- 月度平均正确率
- 最好/最差表现日期
- 整体趋势（改善/恶化）

---

### 场景4: 培训需求识别

**使用页面**: 错误分析 + 实时监控

**操作步骤**:
1. 在错误分析页面查看Top错误类型
2. 在实时监控页面查看重点关注审核人
3. 交叉分析：哪些审核人在哪些错误类型上问题多
4. 制定针对性培训计划

---

### 场景5: 新人上岗跟踪

**使用页面**: 实时监控 + 数据可视化

**操作步骤**:
1. 记录新人上岗日期
2. 每日查看该审核人在"重点关注审核人"中的排名
3. 在数据可视化页面观察正确率变化趋势
4. 评估培训效果

## 常见问题FAQ

### Q1: 页面显示"加载中..."不结束？

**可能原因**:
1. 后端服务未启动
2. 数据库连接失败
3. 选择的日期无数据

**解决方法**:
1. 检查后端服务状态: `curl http://localhost:8000/api/health`
2. 切换到有数据的日期（如 2026-04-01）
3. 联系管理员检查数据库连接

---

### Q2: 为什么某些卡片显示为空？

**原因**: 数据不满足显示条件

**示例**:
- "重点关注队列"为空 → 所有队列正确率 ≥ 90%（这是好事！）
- "高频错误类型"为空 → 当日无错误（这也是好事！）

**这是正常现象，不是Bug**

---

### Q3: 如何导出数据？

**当前版本**: 不支持直接导出

**替代方案**:
1. 截图保存
2. 手动记录关键数据
3. 联系管理员获取原始数据

**未来版本**: 计划支持CSV/Excel导出

---

### Q4: 数据更新频率是多少？

**开发环境**: 
- 点击"刷新数据"按钮手动刷新

## 最佳实践

### 1. 日常使用建议

**每日**:
- 早会前查看实时监控页面
- 关注昨日 vs 前日对比
- 重点关注异常队列和审核人

**每周**:
- 查看错误分析（7天）
- 总结Top错误类型
- 制定下周培训计划

**每月**:
- 查看数据可视化（30天）
- 评估月度质量趋势
- 调整管理策略

### 2. 数据分析思路

1. **先看总体，再看细节**
   - 总览 → 队列 → 审核人

2. **对比分析**
   - 今日 vs 昨日
   - 本周 vs 上周
   - 实际 vs 目标

3. **趋势观察**
   - 短期波动 vs 长期趋势
   - 偶发问题 vs 系统性问题

4. **根因分析**
   - 数据异常 → 查找原因
   - 多维交叉分析
   - 验证假设

### 3. 团队协作建议

**管理者**:
- 每日查看监控数据
- 定期组织数据分析会议
- 跟踪改进措施效果

**质检员**:
- 了解自身表现
- 对比团队平均水平
- 主动提升审核能力

**培训同学**:
- 根据错误数据设计培训
- 跟踪培训效果
- 优化培训内容
