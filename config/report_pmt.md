# 质检日报 PMT（Prompt Template）

> 本文件为日报生成逻辑的固定骨架说明，与 `jobs/daily_report.py` 中的五段式结构一一对应。
> 日报输出逻辑在代码中实现，此文件作为结构参考和维护文档。

## 固定输出格式

```
📊 评论业务质检日报｜{日期}

正确率目标：99.00%

一、今日结论
{一句话总结整体情况，明确是否达标、核心风险点是什么、问题集中在哪个队列}
{若质检量环比波动≥15%，附加提醒：质检量较昨日下降/上升 XX%，请确认是否为业务正常波动}

二、分组表现

【A组】
正确率 {A组正确率} {状态标志}｜质检量 {A组质检量}｜出错 {A组出错量}
结论：{A组判断——稳定到什么程度、是否有观察队列}

【B组】
正确率 {B组正确率} {状态标志}｜质检量 {B组质检量}｜出错 {B组出错量}
其中：B组-评论 {评论正确率} {状态标志}｜B组-账号 {账号正确率} {状态标志}
结论：{B组判断——哪块拖低、风险是否集中}

注：{若某组当日无数据，则在此处说明：B组当日无质检数据}

三、重点风险
1）{最核心风险点，如"XX未达标"}
{正确率数值 + 差距}

2）{问题集中队列}
{是单点集中还是面上扩散 + 归因方向（人员波动/场景识别/标签边界）}

四、处理动作
交付侧：{动作——复盘谁、定位什么}
质培侧：{动作——补什么案例、校准什么口径}
次日关注：{动作——跟踪什么指标}

五、补充风险
告警：P0 {数量} / P1 {数量} / P2 {数量}
判断：{告警与正文风险是否联动、是否已升级为风险事件}
{若无当日告警但昨日有，则显示：昨日告警回顾：P0 X条 / P1 X条 / P2 X条（当日无新增告警）}
```

## 写作原则

1. **站在质培 owner 视角**：先判断，再报数，不要只堆数据
2. **结论先行**：第一句话必须让读者知道"今天行不行、哪里不行"
3. **分组独立判断**：A组稳定不等于没有观察队列，B组达标不等于没有拖累板块
4. **告警联动**：告警不是孤岛，要和正文风险挂钩，判断是否从"波动"升级为"事件"
5. **动作要落地**：区分交付侧和质培侧，明确做"什么"，而不是泛泛的"持续关注"
6. **文风精炼**：适合微信群扫读，每段控制在 2-3 行内
7. **数据完整性**：缺失组必须显式说明，质检量波动超过 15% 必须提醒

## 标志规则

| 条件 | 标志 | 说明 |
|------|------|------|
| 正确率 ≥ 99.00% | ✅ | 达标 |
| 98.50% ≤ 正确率 < 99.00% | 🟡 | 接近达标 |
| 正确率 < 98.50% | 🔴 | 未达标 |

## 状态判断模板

### A组结论
- 整体稳定+无观察队列 → "整体稳定，当前未见明显失控队列，可维持当前审核标准并持续日常监控。"
- 整体稳定+有偏低队列 → "整体稳定，暂未发现明显失控队列。当前偏低队列如「XX」可纳入日常观察。"
- 达标但承压 → "整体达标但承压（XX XX.XX%），非明显失控但需关注局部波动，偏低队列如「XX」需持续观察。"

### B组结论
- 有未达标子业务 → "XX明显拖低表现（XX.XX%，低于目标 X.XXpp），风险集中且方向明确，应优先处理。"
- 多个未达标 → "多个板块未达标，正确率最低为 XX（XX.XX%），需逐板块排查。"

### 告警判断
- P0 > 0 + 存在未达标 → "当日已出现升级风险信号（P0 级），且存在未达标板块，需重点关注是否从「质量波动」升级为「风险事件」。"
- P0 > 0 但大盘达标 → "当日已触发 P0 级告警，需重点关注相关板块后续表现。"
- P1 > 3 → "当日 P1 级告警较多，说明部分指标已偏离正常区间，建议关注。"
- 当日无告警但昨日有 → "昨日告警回顾：P0 X条 / P1 X条 / P2 X条（当日无新增告警）"

## 环比规则

| 指标 | 阈值 | 处理 |
|------|------|------|
| 正确率 | 波动 ≥ 0.3pp | 结论和补充风险中展示 |
| 质检量 | 波动 ≥ 15% | 结论中附加提醒 |

## 配置项

| 配置 | 路径 | 说明 |
|------|------|------|
| 看板链接 | settings.json → dashboard_url | 日报底部展示的看板入口，留空则不显示 |
| 已知业务组 | KNOWN_GROUPS 常量 | 用于检测缺席组，默认 ["A组", "B组"] |
