## 1. 系统概述

**质培运营看板**是一款面向质检团队的数据分析平台，核心功能包括：

| 功能模块 | 核心能力 | 适用人员 |
|---------|---------|---------|
| 📊 **总览** | 核心指标一览、告警监控、组别对比、趋势分析 | 组长、主管 |
| 🔍 **内检** | 跨周期趋势、组别排名、审核人排名、审核一致性 | 质检组长 |
| 🔎 **明细查询** | 多维筛选、问题下钻、数据导出 | 质检员、组长 |
| 👶 **新人追踪** | 新人质量监控、批次分析、个人成长时间线 | 培训负责人 |
| ⚙️ **数据管理** | 导入数据、刷新聚合、维护告警规则 | 数据管理员 |

**技术栈**：Streamlit + TiDB Serverless + GitHub Actions

**访问地址**：部署在 Streamlit Cloud，浏览器访问即可使用。

## 2. 登录与首页

- 打开浏览器，访问看板地址
- 首页（`app.py`）会显示系统状态和导航入口
- 左侧侧边栏可切换不同功能页面：
  - 📊 数据管理
  - 📊 总览
  - 🔍 内检
  - 🔎 明细查询
  - 👶 新人追踪

> **💡 提示**：首次访问或长时间未使用时，页面可能需要 15-30 秒冷启动。这是 Streamlit Cloud 免费版的容器回收机制导致，属正常现象。

## 3. 数据管理

> 入口：侧边栏 → **数据管理**

数据管理页是整个系统的"数据入口"，包含 11 个子功能标签页。

### 3.1 质检数据导入

**操作步骤**：

1. 点击「**质检数据**」标签页
2. 上传质检 Excel 文件（支持 `.xlsx` / `.csv`）
3. 系统自动解析文件名中的日期（如 `2026.4.25长沙云雀质检.xlsx`）
4. 点击确认导入
5. 导入成功后会自动刷新数仓聚合表和告警

**文件命名规范**：
```
日期 + 业务描述 + .xlsx
例：2026.4.25长沙云雀联营评论审核质检.xlsx
例：2026-04-25_内检数据.csv
```

> ⚠️ **注意**：相同日期+来源的数据会被覆盖（幂等导入），不会产生重复。

### 3.2 申诉数据导入

1. 点击「**申诉数据**」标签页
2. 上传申诉 CSV 文件
3. 系统会自动匹配申诉记录到原始质检条目
4. 匹配成功的记录会更新 `is_appeal_reversed` 字段

### 3.3 Google Sheet 导入

1. 点击「**Google Sheet**」标签页
2. 粘贴 Google Sheet 共享链接
3. 系统自动拉取数据（需要 Sheet 设置为"任何人都可查看"）

### 3.4 新人质检数据

1. 点击「**新人质检数据**」标签页
2. 支持批量上传多个新人质检 Excel 文件
3. 上传时会显示实时进度条（第 N/M 个文件）
4. 每个文件导入结果独立显示

### 3.5 新人批次管理

1. 点击「**新人批次管理**」标签页
2. 上传新人名单 Excel（包含姓名、组别、入职日期等字段）
3. 新人名单是"新人追踪"模块的数据基础

### 3.6 新人状态管理

1. 点击「**新人状态管理**」标签页
2. 查看所有新人当前的培训阶段状态
3. 功能包括：
   - **状态扫描**：基于质检数据自动推断每个新人当前处于哪个阶段
   - **晋级推荐**：系统根据可配置的规则自动检测谁满足晋级条件
   - **一键确认/拒绝**：人工审核后一键变更状态

**新人状态流转**：

```
待开始(pending)
   ↓
内检培训(internal_training)
   ↓  连续3天正确率 ≥ 90%
外检培训(external_training)
   ↓  连续3天正确率 ≥ 98%
已毕业(graduated) ← 进入正式队列
```

### 3.7 一键刷新

点击后会依次执行：
1. 刷新数仓聚合表（`mart_day_*`, `mart_week_*`, `mart_month_*`）
2. 刷新告警（根据规则检测异常指标）

### 3.8 告警规则

配置告警触发规则，包括：
- 阈值设置（如正确率 < 95% 触发 P0 告警）
- 告警级别（P0 / P1 / P2）
- 适用范围（指定组别或全局）

### 3.9 其他功能

| 标签页 | 功能 |
|--------|------|
| 上传记录 | 查看历史导入记录（时间、文件、结果） |
| 清除缓存 | 清除 Streamlit 缓存，强制重新加载数据 |
| 清除数据 | 删除指定日期范围的数据（谨慎操作） |

## 4. 总览看板

> 入口：侧边栏 → **总览**

总览页是日常查看的核心页面，提供全局视角的关键指标。

### 4.1 日期和粒度选择

- **粒度**：日 / 周 / 月（页面顶部选择）
- **日期**：选择要查看的具体日期
- 切换后自动刷新所有图表

### 4.2 侧边栏数据状态

打开页面后，侧边栏会显示：
- 🟢 / 🟡 / 🔴 数据新鲜度指示（今日 / T-1 / 延迟 N 天）
- 告警快览（P0 × N · P1 × N）
- 当前筛选范围概要
- 「🔄 刷新缓存」快捷按钮

### 4.3 核心指标概览

顶部展示 4 个核心指标卡片：
- **总质检量**：当前周期质检条目总数
- **整体正确率**：所有组别的加权平均正确率
- **P0 告警数**：当前活跃的高优先级告警数
- **环比变化**：与上一周期对比的变化趋势（↑ / ↓）

### 4.4 实时告警监控

- 展示当前所有活跃告警
- 每条告警显示：级别（P0/P1/P2）、组别、指标、当前值 vs 阈值
- 点击「📋 查看告警详情」展开完整告警表

### 4.5 组别经营视图

- 每个组别一张卡片，展示核心指标
- 点击某个组别卡片，进入该组别的**数据下探分析**：
  - 📈 该组正确率趋势图（近 30 天）
  - 📊 错误类型分布
  - 📋 样本明细（可展开查看）
  
> **💡 提示**：点击趋势图上的数据点，会出现「查看该日明细」按钮，一键跳转到明细查询页并自动带入日期和组别。

### 4.6 质检维度分析

- **质检标签分布**：展示各标签的质检量和正确率
- **质检员工作量**：查看每个质检员的工作负载和正确率
- **仅错误样本分布**：聚焦错误数据，找到高频错误类型

### 4.7 数据导出

页面底部「📥 导出中心」折叠区：
- 导出组别汇总 CSV
- 导出告警列表 CSV
- 导出维度分析 CSV

## 5. 内检分析

> 入口：侧边栏 → **内检**

内检页专注于内部质检数据的深度分析。

### 5.1 日期范围选择

- 在页面顶部选择开始日期和结束日期
- 支持**日维度**、**周维度**、**月维度**三个标签页切换

### 5.2 日维度分析

| 区域 | 内容 |
|------|------|
| 🏆 全量组别排名 | 所有组别按正确率排名的条形图 |
| 📊 组别趋势图 | 各组别正确率的日趋势折线图 |
| 👤 审核人排名 | 展开查看每个审核人的正确率排名 |
| 🏷️ 错误类型 TOP5 | 展开查看错误类型分布和每日变化 |
| 🔄 审核一致性分析 | 展开查看一审 vs 终审偏差散点图 |

### 5.3 审核一致性分析（重点功能）

这是内检页的特色分析模块：

- **散点图**：X 轴=原始正确率，Y 轴=最终正确率
- **气泡大小**：质检量越大，气泡越大
- **颜色**：偏差值（红→绿）
- **对角虚线**：一审=终审的参考线

**如何解读**：
- 点在对角线**上方** → 终审比一审宽松（申诉改判多，一审偏严）
- 点在对角线**下方** → 终审比一审严格（可能存在漏判）
- 偏差 > 2% 的审核人会被自动列出，建议关注

### 5.4 周/月维度

- 周维度和月维度提供更长周期的汇总视图
- 包含组别排名和趋势图
- 适合查看长期变化趋势

## 6. 明细查询

> 入口：侧边栏 → **明细查询**

明细查询页用于查找和导出具体的质检条目。

### 6.1 筛选条件

| 筛选项 | 说明 |
|--------|------|
| 日期范围 | 选择开始/结束日期 |
| 组别 | 下拉选择具体组别，"(全部)"表示不限 |
| 队列 | 下拉选择具体队列 |
| 审核人 | 下拉选择具体审核人 |
| 错误类型 | 下拉选择具体错误类型 |
| 问题筛选 | 原始错误 / 最终错误 / 错判 / 漏判 / 申诉改判 / 全部 |
| 返回条数 | 每次查询返回的最大条数 |

### 6.2 操作按钮

- **🔍 开始查询**：设定好条件后，点击此按钮加载数据
- **🗑️ 重置条件**：一键清除所有筛选条件和上次查询状态

### 6.3 从总览页跳转

当你在总览页点击趋势图数据点后的「查看该日明细」按钮时，会自动跳转到此页面并带入日期和组别条件，自动触发查询。

### 6.4 查询结果

- 表格展示查询结果，支持排序
- 包含：日期、组别、队列、审核人、业务ID、原始判定、最终判定、错误类型等字段
- 底部提供分页功能

### 6.5 数据导出

查询结果下方提供 CSV 下载按钮，可将当前查询结果导出。

## 7. 新人追踪

> 入口：侧边栏 → **新人追踪**

新人追踪页是培训管理的核心工具，监控每位新人的成长曲线。

### 7.1 前置条件

使用前需要在「数据管理 → 新人批次管理」上传新人名单。没有名单时页面会显示提示。

### 7.2 功能视图

页面顶部有多个视图按钮：

| 视图 | 功能 |
|------|------|
| 📊 批次概览 | 查看各批次新人的整体数据 |
| 👤 个人追踪 | 查看单个新人的详细质检趋势和成长时间线 |
| 🚨 告警 | 查看新人相关的告警信息 |
| 🔍 质检明细 | 查看新人质检的原始记录 |

### 7.3 批次概览

- 显示每个培训批次的人数、平均正确率、达标率
- 标记高风险批次（正确率低于阈值）
- 给出团队管理建议

### 7.4 个人追踪

- 选择具体新人后，展示：
  - **状态卡片**：当前培训阶段、累计质检量、正确率
  - **正确率趋势图**：每日正确率折线图
  - **里程碑时间线**：自动记录的关键节点（入职、进入内检、晋级外检、毕业等）

**里程碑时间线**：

```
🟢 2026-03-01  加入培训 (第0天)
    │
🔵 2026-03-15  进入内检培训 (第14天)
    │
🟣 2026-03-30  晋级外检培训 (第29天)
    │
🎉 2026-04-10  正式毕业 (第40天)
```

### 7.5 数据导出

页面底部「📥 数据导出」折叠区提供三种导出：

| 按钮 | 内容 |
|------|------|
| 👥 导出新人名单 | 当前筛选范围内的新人成员 CSV |
| 📊 导出质检数据 | 合并后的质检明细 CSV |
| 📋 导出批次概览 | 批次聚合汇总 CSV |

## 8. 自动报告推送

系统通过 GitHub Actions 自动推送质检报告到企业微信群：

| 报告类型 | 推送时间 | 内容 |
|---------|---------|------|
| 📊 日报 | 每天 18:20（北京时间） | 昨日质检数据汇总、异常指标、AI 洞察 |
| 👶 新人日报 | 每天 18:20（北京时间） | 新人质检表现、风险预警 |
| 📈 周报 | 每周五 18:30 | 一周数据趋势、环比分析、改进建议 |

**手动触发**：在 GitHub 仓库 → Actions → QA Report Pipeline → Run workflow → 选择报告类型和日期。

## 9. 常见问题 FAQ

### Q: 页面加载很慢怎么办？

**A**: 可能的原因和解决方案：

1. **Streamlit Cloud 冷启动**（15-30秒）：长时间未访问后首次打开会较慢，等待即可
2. **数据量大**：总览页采用延迟加载，首屏核心指标会先显示，详情区域点击后才加载
3. **缓存过期**：在侧边栏点击「🔄 刷新缓存」，或在数据管理页使用「清除缓存」
4. **网络问题**：检查网络连接，TiDB Serverless 需要稳定的网络

### Q: 数据没有更新？

**A**: 检查以下步骤：

1. 确认已在「数据管理」页面上传了最新的质检文件
2. 上传后需要等待「刷新数仓」完成
3. 如果使用了缓存，点击侧边栏的「🔄 刷新缓存」
4. 检查侧边栏的数据状态指示器，确认数据日期是否正确

### Q: 导入文件报错？

**A**: 检查以下几点：

1. **文件格式**：确保是 `.xlsx` 或 `.csv` 格式
2. **文件名日期**：文件名中需包含日期信息（如 `2026.4.25xxx.xlsx`）
3. **列名匹配**：确保 Excel 列名与系统预期一致（首行为标题行）
4. **文件编码**：CSV 文件建议使用 UTF-8 编码

### Q: 告警规则如何配置？

**A**: 在「数据管理 → 告警规则」标签页中：

1. 选择要监控的指标（如正确率）
2. 设置阈值和告警级别
3. 选择适用的组别范围
4. 保存后会在下次数据刷新时生效

### Q: 新人状态如何变更？

**A**: 新人状态采用"系统推荐 + 人工确认"的半自动机制：

1. 进入「数据管理 → 新人状态管理」
2. 点击「状态扫描」，系统自动推断每个人的阶段
3. 点击「晋级推荐」，查看满足晋级条件的人员
4. 人工审核后，点击「确认晋级」或「拒绝」

### Q: 如何联系技术支持？

**A**: 如遇到系统问题，请联系数据管理员或在项目 GitHub 仓库提交 Issue。
