## 📊 功能详解

### 1. 问题队列（正确率 < 90%）🔴

**显示内容**:
- 队列名称
- 正确率（红色高亮）
- 审核量

**智能判断**:
- 有问题队列 → 列出所有正确率<90%的队列
- 无问题队列 → 显示"✅ 太棒了！所有队列正确率都 ≥ 90%"

**示例**:
```
🔴 问题队列（正确率 < 90%）         [查看详情 →]

评论_C组       88.50%       审核量: 1,234
弹幕_B组       85.20%       审核量: 987
账号_A组       87.30%       审核量: 765
```

---

### 2. 问题审核人（正确率 < 85%）🟡

**显示内容**:
- 审核人姓名
- 正确率（黄色高亮）
- 审核量 + 误判数

**智能判断**:
- 有问题审核人 → 列出所有正确率<85%的审核人
- 无问题审核人 → 显示"✅ 太棒了！所有审核人正确率都 ≥ 85%"

**示例**:
```
🟡 问题审核人（正确率 < 85%）       [查看详情 →]

张三          78.50%       审核量: 456 | 误判: 12
李四          82.00%       审核量: 389 | 误判: 8
王五          84.20%       审核量: 512 | 误判: 15
```

---

### 3. 高频错误类型（Top 3）🎯

**显示内容**:
- 错误类型名称
- 错误次数（蓝色高亮）
- 占比百分比

**显示逻辑**:
- 始终显示Top 3错误类型
- 按错误数降序排列

**示例**:
```
🎯 高频错误类型（Top 3）            [查看详情 →]

#1 违规引流    45 次        占比: 35.0%
#2 低质导流    28 次        占比: 22.0%
#3 政治敏感    15 次        占比: 12.0%
```

## 📈 性能优化

### 并发加载

- **单个API**: 3 × 200ms = 600ms
- **并发加载**: max(200ms) = 200ms
- **性能提升**: 3倍

### 数据复用

- 问题汇总和下钻模块共享数据
- 避免重复请求
- 内存占用小

### 条件渲染

```typescript
{queueData.length === 0 && (
  <div>暂无数据，点击"查看详情"加载</div>
)}

{queueData.filter(q => q.raw_accuracy_rate < 90).length === 0 && queueData.length > 0 && (
  <div>✅ 太棒了！所有队列正确率都 ≥ 90%</div>
)}
```

**逻辑**:
1. 未加载数据 → 提示点击加载
2. 已加载但无问题 → 显示达标提示
3. 有问题 → 显示问题列表
