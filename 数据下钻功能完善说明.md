# 数据下钻功能完善 - 2026-04-22

## 📋 问题背景

**用户反馈**: "现在所有的下钻都没有 这个合理么？"

**问题分析**:
- 原"数据下钻"区域只有3个静态导航卡片
- 点击后跳转到 `/details` 页面（查询表单）
- **没有真正的下钻功能** ❌
- 用户无法在当前页面查看明细数据

## ✅ 解决方案

### 方案选择: 折叠展开式下钻

**优点**:
- 无需跳转页面，体验流畅
- 按需加载数据，性能优秀
- 操作简单，一目了然

**实现方式**:
- 点击模块标题 → 展开明细数据
- 再次点击 → 收起模块
- 展开时自动调用后端API获取数据

---

## 🚀 实现的功能

### 1. 队列分析下钻

**功能**: 查看各队列明细数据

**数据展示**:
| 字段 | 说明 | 示例 |
|------|------|------|
| 排名 | 按正确率升序 | 1, 2, 3... |
| 队列名称 | queue_name | 评论_A组 |
| 审核量 | qa_cnt | 1,234 |
| 正确率 | raw_accuracy_rate | 99.35% |
| 误判率 | 计算值: 100 - 正确率 | 0.65% |

**特性**:
- 正确率 < 90% 标红 🔴
- 数据按正确率升序（问题队列优先）
- 支持大数字格式化（1,234）

**API**: `GET /api/v1/internal/queues?selected_date=2026-04-01`

---

### 2. 审核人分析下钻

**功能**: 查看审核人表现排名

**数据展示**:
| 字段 | 说明 | 示例 |
|------|------|------|
| 排名 | 按正确率升序 | 1, 2, 3... |
| 审核人 | reviewer_name | 张三 |
| 审核量 | qa_cnt | 456 |
| 正确率 | raw_accuracy_rate | 85.5% |
| 误判数 | misjudge_cnt | 12 |

**特性**:
- 正确率 < 85% 标红 🔴
- 显示误判数而非误判率
- 默认显示Top 20

**API**: `GET /api/v1/internal/reviewers?selected_date=2026-04-01&limit=20`

---

### 3. 错误类型分析下钻

**功能**: 查看高频错误类型

**数据展示**:
| 字段 | 说明 | 示例 |
|------|------|------|
| 排名 | 按错误数降序 | 1, 2, 3... |
| 错误类型 | label_name | 违规引流 |
| 错误数 | cnt | 123 |
| 占比 | pct | 25.6% |

**特性**:
- 显示Top 10错误类型
- 占比自动计算
- 支持"无标签"展示

**API**: `GET /api/v1/internal/error-types?selected_date=2026-04-01&top_n=10`

## 💻 技术实现

**核心Hook**:
```typescript
// 展开状态管理

const [queueData, setQueueData] = useState<any[]>([]);

```

**交互流程**:
1. 用户点击模块标题
2. `toggleModule()` 判断展开/收起
3. 如果是展开 → `loadModuleData()` 调用API
4. 数据加载完成 → 渲染表格

### 后端API

所有API都已存在于 `api/routers/internal.py`：

```python
@router.get("/queues")          # 队列分析
@router.get("/reviewers")       # 审核人分析  
@router.get("/error-types")     # 错误类型分析
```

**返回格式**:
```json
{
  "ok": true,
  "data": {
    "items": [...],
    "total": 10
  }
}
```

**核心样式**:

- `.drill-header`: 可点击的标题栏
- `.drill-content`: 展开的内容区
- `slideDown` 动画: 平滑展开效果

**响应式设计**:
